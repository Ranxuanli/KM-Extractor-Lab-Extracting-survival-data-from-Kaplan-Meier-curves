Workflow

Two ways to run the site

The main page supports both a live extraction path and a safe cached-demo path. For submission screenshots, the cached path is the easiest because it fully complies with the instruction not to upload API keys.

Live extraction

Upload a Kaplan–Meier figure in PNG or JPEG format. If an OpenAI API key is available in the environment, the site sends the image to a multimodal model, requests structured JSON, normalizes the result, then reconstructs pseudo individual patient data and computes approximate survival statistics.

Cached demonstration

This mode simulates a complete successful run using pre-saved extraction output. It is ideal for course submission because the pipeline can be demonstrated through screenshots, cached JSON, and stored output plots with zero risk of exposing credentials.

Recommended screenshot sequence:
  1. Homepage with the polished interface and method description

1. Structured extraction

The model is prompted to return a strict JSON object with a title, axis labels, two treatment arms, approximate sample sizes, and a set of monotone step-curve coordinates.

2. Pseudo-IPD reconstruction

The site converts each KM step function into an approximate event-time table, treating survival drops as events and carrying the remaining subjects forward as censored at the last observed time.

3. Statistical output

From the reconstructed data, the website computes an approximate log-rank test and a one-covariate Cox proportional hazards model to estimate a pairwise hazard ratio with a 95% confidence interval.

Limitations

Why the site reports approximate results

Published Kaplan–Meier figures rarely expose all ingredients required to perfectly reconstruct the original trial data.

Censoring is only partially visible

Unless censoring ticks and risk tables are explicitly extracted, survival drops cannot be perfectly separated into events versus censoring patterns.

Digitization is approximate

Even a strong vision model returns an approximation to the original plotted coordinates. Slight inaccuracies accumulate when transformed into pseudo-IPD.

Use as proof of concept

The output is best presented as a computational methods demonstration for coursework, with clear disclosure that formal inference would require richer underlying data.
